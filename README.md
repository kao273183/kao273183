### What I'm building

Open-source MCP servers for the AI-driven dev pipeline. Each member owns one stage:

```
Idea → Plan → Spec → Code (your IDE) → Test → Coverage → Coach
       mk-plan mk-spec your IDE        mk-qa  mk-spec    both
```

The family wraps the *rails*; code-writing stays in your IDE. Three MCPs, one pipeline.
